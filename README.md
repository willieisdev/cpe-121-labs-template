## How to Submit Your Work

Every lab has a folder. Every assignment inside it is one `.cpp` file. Follow this structure exactly — the auto-grader depends on it.

```
lab1/
    assignment1.cpp
    assignment2.cpp
    assignment3.cpp
    assignment4.cpp
    assignment5.cpp
lab2/
    assignment1.cpp
    ...
```

### Steps

1. Clone or fork this template repository to your own GitHub account.
2. Create the correct folder and file for the assignment you are submitting.
3. Write your solution inside the `.cpp` file.
4. Push to your repository.
5. GitHub Actions will automatically compile, test, and grade your work.
6. Check your score in two places:
   - The **Actions** tab of your repository (click the latest run).
   - The **`RESULT.md`** file that the grader commits back to your repo.

## Grading Breakdown (Per Assignment)

| Component | Marks |
|-----------|-------|
| Code compiles without errors | 8 |
| All test cases pass | 12 |
| **Total per assignment** | **20** |

Each lab has 5 assignments, so each lab is worth **100 marks** total.

---

## Rules

- Your file must be named exactly `assignment1.cpp`, `assignment2.cpp`, etc.
- Your file must be inside the correct folder (`lab1/`, `lab2/`, etc.).
- Do not modify anything inside the `.github/` folder.
- You may push as many times as you like — only the latest score counts.
- Plagiarism detection runs separately. Submitting another student's code will result in zero for the entire lab.
